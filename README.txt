ОТКРОЙ ГОРОД — структура проекта
=================================

СТРАНИЦЫ (открывать прямо двойным кликом в браузере, сервер не нужен):
- index.html    — главная
- events.html   — все мероприятия + фильтр по категориям
- event.html    — карточка одного мероприятия (открывается как event.html?id=gastro)
- cabinet.html  — личный кабинет с бонусами

ПАПКИ:
- css/style.css      — вообще все стили сайта, один файл
- js/data.js         — СПИСОК МЕРОПРИЯТИЙ. Редактируй здесь: название, дата, место,
                        описание, картинка, промокод.
- js/render.js       — как выглядит карточка мероприятия (не трогать без необходимости)
- js/storage.js      — логика бонусов и промокодов (не трогать без необходимости)
- js/common.js       — тема оформления + счётчик бонусов в шапке
- js/page-*.js       — скрипт для конкретной страницы
- assets/images/     — сюда клади картинки мероприятий
- assets/logo/       — сюда клади файл своего логотипа

КАК ДОБАВИТЬ КАРТИНКУ К МЕРОПРИЯТИЮ:
1. Положи файл, например gastro.jpg, в assets/images/
2. Открой js/data.js
3. У нужного мероприятия впиши: image: 'assets/images/gastro.jpg'
Готово — картинка появится и в карточке, и на странице мероприятия.

КАК ЗАМЕНИТЬ ЛОГОТИП:
Сейчас логотип — это SVG-код прямо внутри каждой HTML-страницы (кружок + два
зелёных штриха + текст «ОТКРОЙ ГОРОД»), одинаковый блок в index.html, events.html,
event.html, cabinet.html — все начинаются с <a class="logo" href="index.html">.
Проще всего:
1. Положи готовый файл логотипа (PNG или SVG) в assets/logo/logo.png
2. В КАЖДОМ из 4 HTML-файлов замени блок:
   <svg ...>...</svg>
   на:
   <img src="assets/logo/logo.png" alt="Открой город">
(в css/style.css уже есть стиль .logo img — высота подгонится автоматически)

ЕСЛИ ХОЧЕШЬ ДОБАВИТЬ НОВОЕ МЕРОПРИЯТИЕ:
Просто скопируй один объект в js/data.js внутри массива EVENTS и поменяй поля.
id должен быть уникальным (латиницей, без пробелов) — он используется в ссылках
и в проверке "бонус уже получен".

ДОСТОПРИМЕЧАТЕЛЬНОСТИ (attractions.html):
- Фильтр Музеи / Памятники (js/data.js → SIGHTS)
- Музеи дают бонус по QR так же, как мероприятия (qrEnabled: true) —
  добавь их в qr-codes.html автоматически, страница сама подхватывает SIGHTS
- Памятники — чисто информационные карточки, без QR (qrEnabled: false)

QR-КОДЫ ДЛЯ СТЕНДА (главный "вау-момент" для жюри):
- qr-codes.html — служебная страница, генерирует настоящие сканируемые QR-коды
  для каждого мероприятия. Открой её на ноутбуке/планшете рядом со стендом или
  распечатай кнопкой "Распечатать".
- Участник наводит камеру телефона на QR → попадает на index.html?scan=ID →
  сайт сам зачисляет бонус, играет звук и запускает анимацию звёзд.
- При повторном сканировании того же кода с того же телефона бонус не начислится
  повторно — покажется "вы уже получили бонус".
- Ссылки в QR строятся от текущего адреса страницы, поэтому после публикации на
  GitHub Pages/Netlify/Vercel просто открой qr-codes.html уже по боевому адресу —
  все ссылки в QR пересоберутся автоматически, ничего вручную менять не надо.

МАГАЗИН БОНУСОВ:
- В личном кабинете (cabinet.html) внизу появился блок "Магазин бонусов" —
  участник может обменять накопленные звёзды на призы (кофе, билет, сувенир).
- Список товаров редактируется в js/data.js, массив SHOP: id, title, icon, cost.
- Если бонусов не хватает — кнопка "Обменять" неактивна, а при попытке показывается
  сообщение сколько ещё нужно накопить.
- Траты записываются в ту же историю начислений (со знаком минус) и списываются
  из localStorage, как и начисления.
